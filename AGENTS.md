# AGENTS

Use the repo's actual files and configuration as the source of truth.

- Inspect the repository before changing anything. Read the local docs, relevant code, and config first.
- Keep changes small and surgical. Do not expand scope or perform unrelated cleanup.
- Prefer the simplest approach that fits the evidence in this repo.
- Do not invent commands, tooling, workflows, or architecture that are not present in the repository.
- For non-trivial work, create or update `spec.md` and `todo.md` only when the task benefits from explicit planning and tracking.
- Verify before claiming success. State exactly what you checked and be explicit about anything untested or uncertain.
- Improve existing documentation before adding new files, and avoid repeating the same guidance in multiple places.
